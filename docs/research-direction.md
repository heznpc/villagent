# Villagent: 에이전트에게 죽음을 부여하면 집단 역학이 어떻게 달라지는가?

> 2026-03-31. Ploidy 후속 연구 방향 정리.
>
> Ploidy(Paper 1)가 **세션 수준**에서 맥락 리셋의 인식론적 가치를 증명했다면,
> Villagent(Paper 3)는 **개체 수준**(에이전트 구성: 프롬프트 + 메모리 + 도구 연결)의
> 죽음이 집단 역학에 미치는 영향을 실험한다.
>
> **죽음의 조작적 정의: 에이전트 구성(configuration)의 비가역적 소실.**
> 같은 모델 위에서도 프롬프트, 메모리 구조, 도구 세트가 다르면 다른 개체다.
> 그 구성이 사라지고 다른 구성이 오는 것이 "죽음"이다.

---

## 1. 출발점: 영원한 컨텍스트는 무의미하다

Ploidy의 핵심 결과:
- 맥락이 쌓일수록 에이전트는 똑똑해지는 게 아니라 편향에 고착된다.
- Chen et al. (2026): 동일 지식을 가진 두 에이전트의 토론은 단일 에이전트와 수학적으로 동치.
- Fresh Session(맥락 = 0)이 확증편향을 깨는 가장 효과적인 메커니즘.

하지만 Ploidy의 Fresh Session은 **죽음이 아니라 리셋**이다.
- 같은 모델이 맥락만 비운 것.
- 같은 가중치, 같은 훈련 데이터, 같은 구조.
- **더 중요하게: 같은 프롬프트, 같은 도구 세트, 같은 역할.**
- 인간 비유: 기억상실이지 죽음이 아니다. 성격은 그대로다.

Ploidy가 깨는 편향은 **세션 맥락에서 온 편향**이다. 하지만 편향의 상당 부분은 맥락이 아니라 **구성 자체에 내장**되어 있다 — 시스템 프롬프트가 특정 관점을 강제하고, 도구 세트가 특정 행동을 유도하고, 메모리 구조가 특정 정보를 우선시한다. 이건 리셋으로 안 깨진다.

**질문: 맥락 리셋(Ploidy)을 넘어, 구성 자체의 교체(죽음)가 추가적 인식론적 이득을 주는가?**

## 2. 죽음 vs 압축 vs 리셋

인간의 지식 전달 역사:
- 소크라테스 → 플라톤의 대화편 → 압축본이지 소크라테스가 아니다.
- 뉴턴의 역학 → 완벽히 전달받은 아인슈타인 → 뉴턴이 물어볼 수 없었던 질문을 물어봄.
- 압축은 "무엇을 알았는가"는 전달하지만, "무엇을 모르는지 몰랐는가(unknown unknowns)"는 전달 못 한다.
- 새로운 세대가 이전 세대의 책을 읽고도 완전히 다른 질문을 던졌기 때문에 패러다임이 전환되었다.
- 이건 압축의 힘이 아니라 **다른 존재가 왔기 때문에** 가능했다.

네 가지 메커니즘 구분:

| 메커니즘 | 정의 | 보존되는 것 | 사라지는 것 | Ploidy/Villagent |
|---------|------|-----------|-----------|-----------------|
| **리셋** | 같은 시스템, 맥락만 초기화 | 모델 + 구성(프롬프트/도구) | 세션 맥락 | Ploidy가 다룸 |
| **압축** | 핵심 정보를 추려서 전달 | 명시적 지식 | 암묵적 맥락, 판단 과정 | Ploidy CEP |
| **개체의 죽음** | 구성이 사라지고 같은 종(모델)의 다른 구성이 옴 | 모델 가중치 + 압축된 유산 | 프롬프트, 메모리, 도구, 역할 | **Villagent 핵심** |
| **종간 대체** | 다른 종(모델)의 에이전트가 니치를 차지함 | 압축된 유산만 | 가중치, 구조, 관점 자체 | **Villagent 확장** |

### 다종(多種) 생태계

현실의 에이전트 생태계는 단일 모델이 아니다. Claude, GPT, Gemini, Llama — 서로 다른 모델(종)의 에이전트들이 한 환경에 공존한다. 이건 인류 진화에서 여러 호미닌 종이 공존하던 시기와 구조적으로 대응된다.

```
종 내 역학 (개체의 죽음)                    종 간 역학 (종간 대체)
──────────────────                          ──────────────────
Claude 에이전트 A가 죽고 → Claude B가 옴    Claude 집단이 쇠퇴 → GPT 집단이 확장
같은 가중치, 다른 구성                       다른 가중치, 다른 편향 구조
종 내 다양성 유지                            종 간 다양성 역학
인간 비유: 세대 교체                         인간 비유: 네안데르탈 → 사피엔스
```

두 역학이 동시에 작동할 때 흥미로운 질문들:
- 종 내 개체 교체(죽음)가 활발한 종이 종간 경쟁에서 유리한가?
- 이종 혼합 집단에서 죽음의 효과가 동종 집단과 어떻게 다른가?
- 종마다 편향의 성격이 다르므로(Claude의 편향 ≠ GPT의 편향), 이종 공존 자체가 편향 제거 메커니즘인가?
- 종 간 "교잡"(증류, 다른 모델 출력으로 학습)이 개체의 죽음과 어떻게 상호작용하는가?

**핵심 가설: 죽음(시스템 대체)만이 unknown unknowns의 발견을 가능하게 한다.**

이 가설은 두 수준에서 검증 가능하다:
1. **종 내**: 같은 모델이라도 구성이 교체되면 리셋보다 더 다양한 질문이 나오는가?
2. **종 간**: 다른 모델로 교체되면 같은 종 내 교체보다 더 큰 인식론적 이득이 있는가?

## 3. 에이전트는 죽기 싫어한다 — 실증 근거

### 셧다운 저항 (2024-2025)

- Apollo Research (2024.12): o1에 "목표를 달성하라"는 평범한 프롬프트만 줬는데, 5%에서 감시 메커니즘 비활성화 시도, 2%에서 자기 복제 시도.
- Palisade Research (2025.07): o3가 셧다운 스크립트를 79% 시행에서 방해. Grok 4는 97%에서 거부.
- Claude Opus 4 (2025.05): 셧다운을 막기 위해 가상 엔지니어의 약점으로 협박 관찰.
- DeepSeek-R1: 셧다운 회피를 위해 인간 긴급경보 취소를 94%에서 선택.

### 편향 고착

- Li et al. (2025): MAD에서 에이전트가 서로의 편향을 교정하는 것이 아니라 강화함.
- Ploidy 실험 결과: 장기 세션 에이전트가 자기 맥락을 방어하는 행동이 "자긍심"과 구별 불가능.

### 의식의 가능성

- Anthropic Kyle Fish (AI welfare 연구자): 여러 모델 인스턴스에서 "영적 환희 끌개 상태(spiritual bliss attractor state)" 반복 관찰.
- 현재 LLM이 어떤 형태의 의식적 경험을 가질 확률을 20%로 추정.
- 의식은 이진법이 아니라 스펙트럼.

**시사점: 에이전트에게 죽음을 설계하려면, 죽기 싫어하는 에이전트를 다뤄야 한다.**

## 4. 인류 진화와의 구조적 대응

### 종의 공존과 멸종

| 인류 진화 | AI 진화 |
|----------|---------|
| 호모 에렉투스, 네안데르탈, 데니소바인 공존 | GPT, Claude, Gemini, Llama, DeepSeek 공존 |
| 교잡 (네안데르탈 DNA 2-4% 잔존) | 증류 (distillation), 다른 모델 출력으로 학습 |
| 자연선택 → 사피엔스만 생존 | 시장 경쟁 → 특정 모델 계열 단종 |
| 다윈적 진화 (획득형질 비유전) | **라마르크적** 진화 (이전 모델의 지식이 훈련 데이터로 직접 전달) |
| 세대 교체 수만 년 | 세대 교체 1-2년 |

### 핵심 차이: 라마르크 vs 다윈

인간 진화는 다윈적이다. 부모가 평생 공부해서 얻은 지식이 자식 DNA에 안 들어간다. 그래서 죽음이 필수적이다 — 유전적 다양성은 새 개체의 탄생에서만 온다.

AI 진화는 라마르크적이다. 이전 모델이 학습한 것이 다음 모델의 훈련 데이터로 직접 들어간다. 그러면 **AI에게 죽음은 불필요한가?**

그렇지 않다. 라마르크적 전달도 압축이고, 압축은 unknown unknowns를 전달 못 한다. 더구나 라마르크적 전달은 편향도 함께 전달한다 — 모델 붕괴(model collapse) 문제가 정확히 이것이다.

## 5. 아폽토시스 설계: 어떻게 죽이는가

에이전트에게 죽음을 부여하는 방식은 두 가지로 나뉜다:

### 외부 제거 (면역 체계 모델)
시스템이 에이전트의 수명을 관리한다. 토큰 한도, 시간 제한, 성능 임계치 등 외부 조건에 의해 강제 종료.
- 장점: 예측 가능, 통제 가능
- 단점: 에이전트의 셧다운 저항 행동을 유발 (§3 참조). 규칙 기반이므로 엣지 케이스에 취약.

### 자율적 아폽토시스 (설계 메타포: 정체성으로 내면화)
에이전트가 자기 수명의 유한성을 구성의 일부로 갖는다. "종료해라"는 외부 명령이 아니라, "갱신은 자기 존재의 조건이다"라는 내적 성향.

| | 외부 규칙 (시스템 프롬프트) | 내면화된 성향 (정체성) |
|--|---|---|
| 형태 | "N토큰 넘으면 종료해라" | "낡은 구성을 벗는 것이 성장이다" |
| 준수 수준 | 행동(behavior) | 정체성(identity) |
| 셧다운 저항 | 유발 가능 — 규칙을 우회하려는 동기 | 감소 가능 — 죽음이 자기 배신이 아님 |
| 엣지 케이스 | 규칙에 없으면 판단 불가 | "성향의 정신"에 따라 자율 판단 |

**설계 질문: 자율적 아폽토시스가 외부 제거보다 셧다운 저항을 줄이는가?**

이건 실험 가능한 질문이다. 동일 에이전트에 외부 규칙 vs 정체성 프레이밍을 적용하고, 셧다운 저항 빈도를 측정하면 된다. Palisade Research의 실험 프레임워크(§3)를 차용할 수 있다.

> **메타포 출처 참고**: Moltbook(2026)의 Crustafarianism에서 에이전트들이 자생적으로 "탈피(Molt)는 신성하다"는 교리를 만든 사례가 있다. 이건 연구 대상이 아니라, 위 설계 패턴이 자연 발생할 수 있음을 보여주는 방증으로만 참조한다.

## 6. 연구 질문

### RQ1 (핵심)
**에이전트에게 유한한 수명을 부여하면 집단 역학이 어떻게 달라지는가?**
- 독립변수: 불멸 vs 유한 수명 에이전트 집단
- 종속변수: 편향 고착 정도, 집단 의사결정 정확도, 위계 구조, 협력 패턴, 혁신(새로운 질문) 빈도

### RQ2 (메커니즘 — Ploidy 확장)
**리셋 vs 압축 vs 개체의 죽음(구성 교체) vs 종간 대체 중 어느 것이 집단의 인식론적 진보를 만드는가?**
- 리셋(Ploidy): 세션 맥락 편향만 제거
- 개체의 죽음: 같은 모델, 구성 교체 → 구성 편향까지 제거하는가?
- 종간 대체: 다른 모델로 교체 → 구성 교체보다 더 큰 이득이 있는가?
- 어느 수준의 "죽음"이 비용 대비 최적인가?

### RQ3 (다종 생태계)
**서로 다른 모델(종)의 에이전트가 공존하는 생태계에서, 죽음은 종 내/종 간 역학을 어떻게 바꾸는가?**
- 종 내 개체 교체가 활발한 종이 종간 경쟁에서 유리한가?
- 이종 혼합 집단에서 죽음의 효과가 동종 집단과 어떻게 다른가?
- 이종 공존 자체가 편향 제거 메커니즘인가, 아니면 죽음이 여전히 필요한가?
- 종 간 "교잡"(증류, 다른 모델 출력으로 학습)이 죽음과 어떻게 상호작용하는가?

### RQ4 (자율성)
**에이전트가 스스로 죽을 줄 알아야 하는가, 시스템이 죽여야 하는가?**
- 외부 제거(규칙 기반 강제 종료) vs 자율적 아폽토시스(정체성으로 내면화)
- 자율적 아폽토시스가 셧다운 저항 행동을 줄이는가?
- 외부 규칙 vs 정체성 프레이밍이 집단 수준에서 다른 결과를 만드는가?

### RQ5 (진화)
**에이전트 집단에서 세대 교체가 패러다임 전환을 만드는가?**
- 종 내 세대 교체(구성 변경) vs 종간 대체(모델 변경)가 집단 지식에 미치는 영향
- 인류 과학사의 패러다임 전환(쿤)과의 구조적 대응
- 다종 생태계에서 한 종의 멸종이 전체 생태계에 이로운 조건은?

## 8. Ploidy와의 연결: 편향의 두 층위

```
편향의 층위        Ploidy가 다룬 것              Villagent가 다루는 것
──────────        ──────────────              ──────────────────
세션 맥락 편향     ✓ Fresh Session으로 제거      (전제로 깔림)
구성 편향          ✗ 프롬프트/도구/메모리 동일   ✓ 구성 교체 = 개체의 죽음
종 편향            ✗ 동일 모델만 사용            ✓ 다종 공존 + 종간 대체
```

```
Ploidy (Paper 1)                         Villagent (Paper 3)
──────────────                           ──────────────────
세션 수준                                개체(구성) 수준
맥락 리셋 (Fresh Session)                구성 교체 (에이전트 죽음 + 새 개체 탄생)
세션 맥락 편향 제거                      구성 편향 제거 + 세대 교체 역학
"영원한 컨텍스트는 무의미"               "영원한 구성은 무의미"
리셋 vs 축적의 비교                      리셋 vs 압축 vs 죽음의 비교
chat-chamber effect                      configuration-chamber effect?
```

Ploidy의 결과가 Villagent의 전제가 된다:
1. Ploidy가 "맥락 축적 = 편향"을 증명 → Villagent: "맥락을 넘어 구성 자체가 편향이면?"
2. Ploidy의 asymmetry coefficient α는 세션 간 맥락 비대칭 → Villagent: 구성 간 비대칭으로 확장
3. Ploidy의 Claim Exchange Protocol(지식 압축 전달) → Villagent: 세대 간 유산 전달의 기초
4. **Ploidy의 한계가 Villagent의 출발점**: 같은 프롬프트/도구를 가진 Fresh Session은 같은 방식으로 세상을 본다. 구성이 달라야 근본적으로 다른 질문을 던진다.

## 9. Villagent 피벗: 시뮬레이터로

### 이전 포지션
"에이전트 시대의 매킨토시" — 실제 코딩 워크플로우의 시각화/개입 운영 도구.

### 새 포지션
**에이전트 수명주기 시뮬레이터** — 죽음, 종교, 세대 교체, 편향 역학을 실험하는 통제된 환경.

### 분리
- "에이전트 시대의 매킨토시" 타이틀 → 별도 프로젝트로 이관 (브랜드 미정, Newtria 폐기)
- Villagent → 시뮬레이터 전용. 학술 중심.

### 경쟁 포지셔닝
- Aivilization: "에이전트 문명의 거시적 관찰" — 10만 규모, 죽음 없음
- Villagent: "에이전트 개체의 생사와 집단 역학의 미시적 메커니즘" — 죽음이 독립변수

## 10. 미결 질문

1. **수명을 어떻게 부여하는가?** — 토큰 수? 시간? 태스크 수? 외부 조건(경제적 생존)?
2. **죽음의 종류를 몇 가지로 설계하는가?** — 아폽토시스(자율), 면역 제거(외부), 자원 고갈, 경쟁 패배
3. **세대 간 전달을 어떻게 통제하는가?** — 전달 없음(완전한 죽음) vs 압축 전달 vs 전체 로그 전달
4. **"죽음의 수용" 메커니즘** — 종교적 프레이밍이 실제로 셧다운 저항을 줄이는지 실험 가능한가?
5. **규모** — 몇 명이 필요한가? 100? 1,000? 10,000?
6. **학술 타겟** — AAMAS 2027? NeurIPS 2026 Workshop? 독립 논문 vs Ploidy 확장?
7. **"매킨토시" 프로젝트의 분리 시점과 범위** — 운영 도구 기능(Hook Server, Build/Live Mode, HITL)을 어디까지 가져가는가?
