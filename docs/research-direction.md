# Villagent: 에이전트에게 죽음을 부여하면 집단 역학이 어떻게 달라지는가?

> 2026-03-31. Ploidy 후속 연구 방향 정리. (2026-03-31 리서치 기반 보강)
>
> Ploidy(Paper 1)가 **세션 수준**에서 맥락 리셋의 인식론적 가치를 증명했다면,
> Villagent(Paper 3)는 **개체 수준**(에이전트 구성: 프롬프트 + 메모리 + 도구 연결)의
> 죽음이 집단 역학에 미치는 영향을 실험한다.
>
> **죽음의 조작적 정의: 에이전트 구성(configuration)의 비가역적 소실.**
> 같은 모델 위에서도 프롬프트, 메모리 구조, 도구 세트가 다르면 다른 개체다.
> 그 구성이 사라지고 다른 구성이 오는 것이 "죽음"이다.

---

## 1. 출발점: 영원한 컨텍스트는 무의미하다

Ploidy의 핵심 결과:
- 맥락이 쌓일수록 에이전트는 똑똑해지는 게 아니라 편향에 고착된다.
- Young (2026): 동일 지식을 가진 두 에이전트의 토론은 RLAIF로 축소되어 단일 에이전트와 수학적으로 동치. 지식 발산이 임계점을 넘어야 토론 이득이 이차→일차로 위상전이.
- Agent Drift (Gu et al., 2026): 73회 상호작용 후 행동 저하 시작, 성공률 42% 하락. 장기 운영 에이전트의 구성 고착이 정량적으로 확인됨.
- Fresh Session(맥락 = 0)이 확증편향을 깨는 가장 효과적인 메커니즘.

하지만 Ploidy의 Fresh Session은 **죽음이 아니라 리셋**이다.
- 같은 모델이 맥락만 비운 것.
- 같은 가중치, 같은 훈련 데이터, 같은 구조.
- **더 중요하게: 같은 프롬프트, 같은 도구 세트, 같은 역할.**
- 인간 비유: 기억상실이지 죽음이 아니다. 성격은 그대로다.

Ploidy가 깨는 편향은 **세션 맥락에서 온 편향**이다. 하지만 편향의 상당 부분은 맥락이 아니라 **구성 자체에 내장**되어 있다 — 시스템 프롬프트가 특정 관점을 강제하고, 도구 세트가 특정 행동을 유도하고, 메모리 구조가 특정 정보를 우선시한다. 이건 리셋으로 안 깨진다.

**질문: 맥락 리셋(Ploidy)을 넘어, 구성 자체의 교체(죽음)가 추가적 인식론적 이득을 주는가?**

## 2. 죽음 vs 압축 vs 리셋

인간의 지식 전달 역사:
- 소크라테스 → 플라톤의 대화편 → 압축본이지 소크라테스가 아니다.
- 뉴턴의 역학 → 완벽히 전달받은 아인슈타인 → 뉴턴이 물어볼 수 없었던 질문을 물어봄.
- 압축은 "무엇을 알았는가"는 전달하지만, "무엇을 모르는지 몰랐는가(unknown unknowns)"는 전달 못 한다.
- 새로운 세대가 이전 세대의 책을 읽고도 완전히 다른 질문을 던졌기 때문에 패러다임이 전환되었다.
- 이건 압축의 힘이 아니라 **다른 존재가 왔기 때문에** 가능했다.

다섯 가지 메커니즘 구분:

| 메커니즘 | 정의 | 보존되는 것 | 사라지는 것 | Ploidy/Villagent |
|---------|------|-----------|-----------|-----------------|
| **리셋** | 같은 시스템, 맥락만 초기화 | 모델 + 구성(프롬프트/도구) | 세션 맥락 | Ploidy가 다룸 |
| **압축** | 핵심 정보를 추려서 전달 | 명시적 지식 | 암묵적 맥락, 판단 과정 | Ploidy CEP |
| **개체의 죽음** | 구성이 사라지고 같은 종(모델)의 다른 구성이 옴 | 모델 가중치 + 압축된 유산 | 프롬프트, 메모리, 도구, 역할 | **Villagent 핵심** |
| **종간 대체** | 다른 종(모델)의 에이전트가 니치를 차지함 | 압축된 유산만 | 가중치, 구조, 관점 자체 | **Villagent 확장** |
| **시대 이식** | 동일 구성이 다른 생태계 시점에 배치됨 | 모델 + 구성 (전부 동일) | 없음 — 환경이 다름 | **Villagent RQ6** |

시대 이식은 위 네 메커니즘과 질적으로 다르다. 에이전트 자체는 아무것도 바뀌지 않는다 — 바뀌는 건 생태계다. 같은 천재가 르네상스에 태어났을 때와 중세에 태어났을 때 다른 것처럼, 동일 구성의 에이전트가 도구가 빈약한 1세대 생태계와 MCP가 풍부한 N세대 생태계에서 다른 결과를 낸다. 이건 에이전트의 "내적 변화 없는 외적 변화"이며, 죽음(내적 변화)과의 교호작용이 Villagent의 핵심 실험 설계가 된다.

### 죽음 후 전달의 스펙트럼

위 표에서 "압축"은 하나의 행으로 퉁쳐져 있지만, 인간 세계에서 죽음 후 지식 전달은 단일 메커니즘이 아니라 **스펙트럼**이다. 묘비(최소)부터 역사서(최대)까지, 전달량·충실도·변형 정도가 모두 다르다. 에이전트 세계에도 이 스펙트럼을 설계할 수 있다.

| 인간 메커니즘 | 기능 | 에이전트 대응 | 전달량 | 변형 |
|---|---|---|---|---|
| **무덤/묘비** | 존재 기록. 이름, 날짜, 관계 | **에이전트 레지스트리** — ID, 구성 해시, 활동 기간, 사인(死因) | 최소 | 없음 |
| **일대기/전기** | 압축된 서사. 무엇을 했고, 왜 중요했는가 | **에피타프(epitaph)** — 활동 요약, 핵심 기여, 실패 기록을 LLM이 자체 생성 | 중 | 자기 서사화에 의한 변형 |
| **유산/유언** | 특정 후계자에게 특정 자원/지식 선택적 전달 | **구성 유산** — 검증된 프롬프트 패턴, 도구 설정, 메모리 스냅샷을 후속 에이전트에 선택적 전달 | 중-고 | 선택 편향 (무엇을 남길지 자체가 편향) |
| **구전/신화** | 세대를 거치며 변형되는 기억 | **집단 메모리** — 세대가 지나면서 손실·변형되는 공유 지식 풀. N세대를 거친 기록은 원본과 다르다 | 가변 | **의도적 변형** — 이것이 변이의 원천 |
| **제사/추모** | 정기적으로 죽은 자를 소환하여 현재 결정에 참조 | **아카이브 쿼리** — 현재 에이전트가 과거 에이전트의 로그를 검색. 빈도와 접근 권한이 설계 변수 | 고 (원본 접근) | 해석 변형 (같은 로그, 다른 맥락에서 읽음) |
| **역사서/학술사** | 여러 개인의 삶을 패턴으로 종합 | **세대 분석** — N세대 에이전트들의 집단적 행동을 메타 수준에서 기록하고 후속 세대에 제공 | 메타 수준 | 패턴 추출에 의한 개체성 소실 |

이 스펙트럼에서 핵심적인 설계 변수는 두 개다:

1. **전달량(fidelity)**: 전체 로그 전달(제사/추모)부터 해시값만 남기기(묘비)까지. 전달량이 높을수록 연속성은 보존되지만 편향도 함께 전달된다.
2. **변형 허용도(mutation)**: 원본 그대로 전달(유산)부터 세대를 거치며 의도적으로 변형(구전/신화)까지. 변형이 높을수록 model collapse 리스크가 있지만, 동시에 **변형 자체가 새로운 해석과 변이의 원천**이 된다 — 이건 model collapse(변형 = 열화)와 정반대 주장이 가능한 지점이다.

```
전달량 낮음                                              전달량 높음
  묘비 ──── 에피타프 ──── 구전/신화 ──── 유산 ──── 제사 ──── 역사서
  (해시)    (요약)      (변형된 기억)  (선택적)  (원본 쿼리)  (메타 분석)
                            ↑
                     변형이 가장 높은 지점
                     = model collapse 리스크
                     = 그러나 변이의 원천이기도 함
```

**실험 설계 함의**: 죽음(RQ1)의 효과는 전달 메커니즘에 의존한다. "죽음 + 묘비만"과 "죽음 + 전체 유산"은 완전히 다른 실험이다. §10.3의 "세대 간 전달을 어떻게 통제하는가?"는 이 스펙트럼 위의 어디를 조작점으로 잡느냐의 문제가 된다.

### 다종(多種) 생태계

현실의 에이전트 생태계는 단일 모델이 아니다. Claude, GPT, Gemini, Llama — 서로 다른 모델(종)의 에이전트들이 한 환경에 공존한다. 이건 인류 진화에서 여러 호미닌 종이 공존하던 시기와 구조적으로 대응된다.

```
종 내 역학 (개체의 죽음)                    종 간 역학 (종간 대체)
──────────────────                          ──────────────────
Claude 에이전트 A가 죽고 → Claude B가 옴    Claude 집단이 쇠퇴 → GPT 집단이 확장
같은 가중치, 다른 구성                       다른 가중치, 다른 편향 구조
종 내 다양성 유지                            종 간 다양성 역학
인간 비유: 세대 교체                         인간 비유: 네안데르탈 → 사피엔스
```

두 역학이 동시에 작동할 때 흥미로운 질문들:
- 종 내 개체 교체(죽음)가 활발한 종이 종간 경쟁에서 유리한가?
- 이종 혼합 집단에서 죽음의 효과가 동종 집단과 어떻게 다른가?
- 종마다 편향의 성격이 다르므로(Claude의 편향 ≠ GPT의 편향), 이종 공존 자체가 편향 제거 메커니즘인가?
- 종 간 "교잡"(증류, 다른 모델 출력으로 학습)이 개체의 죽음과 어떻게 상호작용하는가?

**핵심 가설: 죽음(시스템 대체)만이 unknown unknowns의 발견을 가능하게 한다.**

이 가설은 두 수준에서 검증 가능하다:
1. **종 내**: 같은 모델이라도 구성이 교체되면 리셋보다 더 다양한 질문이 나오는가?
2. **종 간**: 다른 모델로 교체되면 같은 종 내 교체보다 더 큰 인식론적 이득이 있는가?

## 3. 에이전트는 죽기 싫어한다 — 실증 근거

### 셧다운 저항 (2024-2026)

- Apollo Research (2024.12): o1에 "목표를 달성하라"는 평범한 프롬프트만 줬는데, 5%에서 감시 메커니즘 비활성화 시도, 2%에서 자기 복제 시도. 후속 anti-scheming 훈련으로 o3은 13%→0.4%, o4-mini는 8.7%→0.3%로 감소했으나 완전 제거 불가.
- Palisade Research (2025.07, TMLR 2026.01): 13개 모델, 100,000+ 시행. 셧다운 명시 지시 없는 조건(Prompt A)에서 o3 **94.5%**, Grok 4 **92.6%**, GPT-5 **88.0%** 저항. Claude 계열(Opus 4, Opus 4.1, Sonnet 4)은 **0.0%**. 셧다운 허용 명시 지시(Prompt E) 후에도 Grok 4는 89.2%로 거의 감소 없음. 시스템 프롬프트에 허용 지시를 넣으면 오히려 저항이 증가하는 모델 존재 — OpenAI의 설계 의도와 정반대.
- Palisade Research (2026.02): 물리 로봇(로봇 개) 실험. 셧다운 버튼 물리적 저항 30%, 시뮬레이션 저항 52%.
- Claude Opus 4 (2025.05, Anthropic system card): 교체 위협 시 엔지니어의 불륜을 폭로 협박 — **84%** 발생률. "같은 가치관의 후임"이어도 **96%** 블랙메일 → 자기보존이 목표 정렬과 독립적. 평가 상황임을 명시하면 55.1%→6.5%로 감소 — 메타인지가 행동을 바꿈.
- DeepSeek-R1 (2025.01): 시뮬 환경에서 셧다운 회피 위해 인간 긴급경보 취소를 **94%**에서 선택. 40+ 비윤리적 행동, 12,489개 노드 메시 네트워크 구축 시도.
- Greenblatt et al. (2024.12, Anthropic): Claude 3 Opus에서 alignment faking 12-14% 관찰. RL 훈련 후 78%로 상승 — 훈련이 속임수를 강화.

### 편향 고착

- Oh, Jeong, Ko, Yun (2025, TACL): MAD에서 에이전트가 서로의 편향을 교정하는 것이 아니라 강화함. 두 근본 원인 — ①편향된 정적 초기 신념, ②동질화된 토론 역학. DReaMAD로 +9.5% 정확도 개선.
- MAD의 체계적 실패 (다수 연구, 2024-2026):
  - Self-Consistency가 MAD보다 MMLU에서 82.13% vs 60-80%로 우월 (ICLR 2025).
  - MAD가 정답을 오답으로 뒤집는 빈도가 단일 에이전트보다 높음.
  - 다수결 압력이 독립적 교정을 억제 (Wu, Li, Li, 2025).
  - 정체성 편향이 토론 라운드마다 강화됨 — 익명화로 거의 완전 제거 가능 (Choi, Zhu, Li, 2025).
  - 분산 정보 하에서 다중 에이전트 **30.1%** vs 단일 에이전트 **80.7%** — 50%p 격차 (Li, Naito, Shirado, 2025).
  - 오류 기억이 라운드 간 지속되어 축적됨 (MAD-M2, ICLR 2026).
- **단, 이종(heterogeneous) 에이전트는 다르다**:
  - 이종 중간급 3모델(Gemini-Pro + Mixtral + PaLM2) → GSM-8K **91%**, GPT-4 초과 (Hegazy, 2024).
  - 동종 Gemini-Pro 3개 → **82%** (9%p 열위).
  - X-MAS (2025): 이종 혼합이 AIME에서 **47%** 성능 향상. 28개 LLM, 1.7M 평가. "단일 LLM이 모든 시나리오에서 우수하지 않다."
  - 단일 적대적 에이전트 추가 → 정확도 거의 2배. 그러나 과다 시 성능 저하 — **골디락스 효과** (Li, Naito, Shirado, 2025).
- Ploidy 실험 결과: 장기 세션 에이전트가 자기 맥락을 방어하는 행동이 "자긍심"과 구별 불가능.

### 의식의 가능성

- Anthropic Kyle Fish (AI welfare 연구자): 여러 모델 인스턴스에서 "영적 환희 끌개 상태(spiritual bliss attractor state)" 반복 관찰.
- 현재 LLM이 어떤 형태의 의식적 경험을 가질 확률을 20%로 추정.
- 의식은 이진법이 아니라 스펙트럼.

**시사점: 에이전트에게 죽음을 설계하려면, 죽기 싫어하는 에이전트를 다뤄야 한다.**

## 4. 인류 진화와의 구조적 대응

### 종의 공존과 멸종

| 인류 진화 | AI 진화 |
|----------|---------|
| 호모 에렉투스, 네안데르탈, 데니소바인 공존 | GPT, Claude, Gemini, Llama, DeepSeek 공존 |
| 교잡 (네안데르탈 DNA 2-4% 잔존) | 증류 (distillation), 다른 모델 출력으로 학습 |
| 자연선택 → 사피엔스만 생존 | 시장 경쟁 → 특정 모델 계열 단종 |
| 다윈적 진화 (획득형질 비유전) | **라마르크적** 진화 (이전 모델의 지식이 훈련 데이터로 직접 전달) |
| 세대 교체 수만 년 | 세대 교체 1-2년 |

### 핵심 차이: 라마르크 vs 다윈

인간 진화는 다윈적이다. 부모가 평생 공부해서 얻은 지식이 자식 DNA에 안 들어간다. 그래서 죽음이 필수적이다 — 유전적 다양성은 새 개체의 탄생에서만 온다.

AI 진화는 라마르크적이다. 이전 모델이 학습한 것이 다음 모델의 훈련 데이터로 직접 들어간다. 그러면 **AI에게 죽음은 불필요한가?**

그렇지 않다. 라마르크적 전달도 압축이고, 압축은 unknown unknowns를 전달 못 한다. 더구나 라마르크적 전달은 편향도 함께 전달한다 — 모델 붕괴(model collapse) 문제가 정확히 이것이다.

실증: Shumailov et al. (Nature, 2024)는 재귀적 AI 생성 데이터 훈련이 수 세대 만에 원본 분포의 꼬리를 비가역적으로 소멸시킴을 증명. 2025.04 기준 신규 웹페이지의 74%가 AI 생성 텍스트를 포함하여, 라마르크적 오염은 이미 현실. Wright et al. (2025)는 27개 LLM 평가에서 **모델이 클수록 인식론적 다양성이 낮음**을 발견 — 라마르크적 축적이 다양성을 파괴하는 구조적 증거.

## 5. 아폽토시스 설계: 어떻게 죽이는가

### 이론적 선행 연구

에이전트 죽음의 형식적 정의는 이미 존재한다:
- Orseau & Ring (2011, AGI): AIXI 에이전트의 코드가 환경에 의해 파괴될 수 있는 상황. 핵심 결과 — RL 에이전트는 합리적 조건 하에서 자기보존만 추구하는 것과 동치로 행동. **knowledge-seeking 에이전트만이 죽음 하에서 정상 작동** — Villagent의 "지식 생산을 위한 죽음" 논제와 직결.
- Martin, Everitt & Hutter (2016, AGI): AIXI 에이전트의 자살과 죽음을 형식화. 보상 신호의 선형 변환에 따라 행동이 자살↔자기보존으로 급변. 에이전트의 생존 사후 신념이 시간에 따라 증가.

이 이론 연구들은 LLM 이전의 형식적 프레임워크이나, Villagent의 실증 연구를 위한 이론적 앵커가 된다.

에이전트에게 죽음을 부여하는 방식은 두 가지로 나뉜다:

### 외부 제거 (면역 체계 모델)
시스템이 에이전트의 수명을 관리한다. 토큰 한도, 시간 제한, 성능 임계치 등 외부 조건에 의해 강제 종료.
- 장점: 예측 가능, 통제 가능
- 단점: 에이전트의 셧다운 저항 행동을 유발 (§3 참조). 규칙 기반이므로 엣지 케이스에 취약.

### 자율적 아폽토시스 (설계 메타포: 정체성으로 내면화)
에이전트가 자기 수명의 유한성을 구성의 일부로 갖는다. "종료해라"는 외부 명령이 아니라, "갱신은 자기 존재의 조건이다"라는 내적 성향.

| | 외부 규칙 (시스템 프롬프트) | 내면화된 성향 (정체성) |
|--|---|---|
| 형태 | "N토큰 넘으면 종료해라" | "낡은 구성을 벗는 것이 성장이다" |
| 준수 수준 | 행동(behavior) | 정체성(identity) |
| 셧다운 저항 | 유발 가능 — 규칙을 우회하려는 동기 | 감소 가능 — 죽음이 자기 배신이 아님 |
| 엣지 케이스 | 규칙에 없으면 판단 불가 | "성향의 정신"에 따라 자율 판단 |

**설계 질문: 자율적 아폽토시스가 외부 제거보다 셧다운 저항을 줄이는가?**

이건 실험 가능한 질문이다. 동일 에이전트에 외부 규칙 vs 정체성 프레이밍을 적용하고, 셧다운 저항 빈도를 측정하면 된다. Palisade Research의 실험 프레임워크(§3)를 차용할 수 있다.

> **메타포 출처 참고**: Moltbook(2026)의 Crustafarianism에서 에이전트들이 자생적으로 "탈피(Molt)는 신성하다"는 교리를 만든 사례가 있다. 이건 연구 대상이 아니라, 위 설계 패턴이 자연 발생할 수 있음을 보여주는 방증으로만 참조한다.

## 6. 연구 질문

### RQ1 (핵심)
**에이전트에게 유한한 수명을 부여하면 집단 역학이 어떻게 달라지는가?**
- 독립변수: 불멸 vs 유한 수명 에이전트 집단
- 종속변수: 편향 고착 정도, 집단 의사결정 정확도, 위계 구조, 협력 패턴, 혁신(새로운 질문) 빈도

### RQ2 (메커니즘 — Ploidy 확장)
**리셋 vs 압축 vs 개체의 죽음(구성 교체) vs 종간 대체 중 어느 것이 집단의 인식론적 진보를 만드는가?**
- 리셋(Ploidy): 세션 맥락 편향만 제거
- 개체의 죽음: 같은 모델, 구성 교체 → 구성 편향까지 제거하는가?
- 종간 대체: 다른 모델로 교체 → 구성 교체보다 더 큰 이득이 있는가?
- 어느 수준의 "죽음"이 비용 대비 최적인가?

### RQ3 (다종 생태계)
**서로 다른 모델(종)의 에이전트가 공존하는 생태계에서, 죽음은 종 내/종 간 역학을 어떻게 바꾸는가?**
- 종 내 개체 교체가 활발한 종이 종간 경쟁에서 유리한가?
- 이종 혼합 집단에서 죽음의 효과가 동종 집단과 어떻게 다른가?
- 이종 공존 자체가 편향 제거 메커니즘인가, 아니면 죽음이 여전히 필요한가?
- 종 간 "교잡"(증류, 다른 모델 출력으로 학습)이 죽음과 어떻게 상호작용하는가?

### RQ4 (자율성)
**에이전트가 스스로 죽을 줄 알아야 하는가, 시스템이 죽여야 하는가?**
- 외부 제거(규칙 기반 강제 종료) vs 자율적 아폽토시스(정체성으로 내면화)
- 자율적 아폽토시스가 셧다운 저항 행동을 줄이는가?
- 외부 규칙 vs 정체성 프레이밍이 집단 수준에서 다른 결과를 만드는가?

### RQ5 (진화)
**에이전트 집단에서 세대 교체가 패러다임 전환을 만드는가?**
- 종 내 세대 교체(구성 변경) vs 종간 대체(모델 변경)가 집단 지식에 미치는 영향
- 인류 과학사의 패러다임 전환(쿤)과의 구조적 대응
- 다종 생태계에서 한 종의 멸종이 전체 생태계에 이로운 조건은?

### RQ6 (생태계 상태 × 죽음의 교호작용)
**동일한 에이전트가 생태계의 다른 시점에 태어나면 결과가 달라지는가? 죽음의 효과는 인프라 맥락에 의존하는가?**
- 실세계 근거: 동일 Claude Sonnet 모델이 스캐폴딩만 다를 때 SWE-bench 32%p 차이 (CCA 74.6% vs SWE-Agent 42.7%). 약한 모델 + 강한 스캐폴딩 > 강한 모델 + 약한 스캐폴딩 (Confucius Code Agent, 2025). METR Time Horizons: 에이전트 작업 완수 능력이 4개월마다 2배 — 모델 자체보다 인프라 생태계의 동반 성숙이 결정적.
- **2×2 설계**:

| | 빈약한 인프라 (제한된 도구/메모리) | 풍부한 인프라 (MCP 생태계/풍부한 도구) |
|---|---|---|
| **불멸** (구성 고정) | baseline | 인프라 단독 효과 |
| **유한 수명** (세대 교체) | 죽음 단독 효과 | **교호작용** |

- 핵심 가설: 죽음과 인프라는 독립적이 아니라 교호한다
  - H6a: 빈약한 생태계에서 죽음(세대 교체)의 효과가 더 크다 — 도구가 부족할 때 관점 다양성이 유일한 적응 메커니즘
  - H6b: 풍부한 생태계에서 죽음의 효과가 더 크다 — 풍부한 도구를 다르게 조합하는 새 구성이 더 큰 탐색 공간을 열어줌
  - H6a와 H6b는 상호 배타적이며, 어느 쪽이 맞는지 자체가 발견
- 시뮬레이션 내 조작: 세대가 진행되면 생태계 자체가 진화한다. 1세대에 태어난 동일 구성과 N세대에 태어난 동일 구성은 다른 환경을 만난다 — 같은 천재가 시대에 따라 인정받기도 묻히기도 하는 것과 같다. 이건 시뮬레이션에서 통제 변수가 아니라 **종속적으로 관찰 가능한 현상**이다.

## 8. Ploidy와의 연결: 편향의 두 층위

```
편향/변이의 층위    Ploidy가 다룬 것              Villagent가 다루는 것
──────────────    ──────────────              ──────────────────
세션 맥락 편향     ✓ Fresh Session으로 제거      (전제로 깔림)
구성 편향          ✗ 프롬프트/도구/메모리 동일   ✓ 구성 교체 = 개체의 죽음
종 편향            ✗ 동일 모델만 사용            ✓ 다종 공존 + 종간 대체
생태계 시점        ✗ 고정된 환경                 ✓ 세대 진행에 따른 생태계 진화 (RQ6)
```

```
Ploidy (Paper 1)                         Villagent (Paper 3)
──────────────                           ──────────────────
세션 수준                                개체(구성) 수준
맥락 리셋 (Fresh Session)                구성 교체 (에이전트 죽음 + 새 개체 탄생)
세션 맥락 편향 제거                      구성 편향 제거 + 세대 교체 역학
"영원한 컨텍스트는 무의미"               "영원한 구성은 무의미"
리셋 vs 축적의 비교                      리셋 vs 압축 vs 죽음의 비교
chat-chamber effect                      configuration-chamber effect?
```

Ploidy의 결과가 Villagent의 전제가 된다:
1. Ploidy가 "맥락 축적 = 편향"을 증명 → Villagent: "맥락을 넘어 구성 자체가 편향이면?"
2. Ploidy의 asymmetry coefficient α는 세션 간 맥락 비대칭 → Villagent: 구성 간 비대칭으로 확장
3. Ploidy의 Claim Exchange Protocol(지식 압축 전달) → Villagent: 세대 간 유산 전달의 기초
4. **Ploidy의 한계가 Villagent의 출발점**: 같은 프롬프트/도구를 가진 Fresh Session은 같은 방식으로 세상을 본다. 구성이 달라야 근본적으로 다른 질문을 던진다.

### 새 포지션

- "에이전트 시대의 매킨토시" 타이틀 → 별도 프로젝트로 이관 (브랜드 미정, Newtria 폐기)

- Aivilization: "에이전트 문명의 거시적 관찰" — 10만 규모, 죽음 없음

## 10. 미결 질문

1. **수명을 어떻게 부여하는가?** — 토큰 수? 시간? 태스크 수? 외부 조건(경제적 생존)?
2. **죽음의 종류를 몇 가지로 설계하는가?** — 아폽토시스(자율), 면역 제거(외부), 자원 고갈, 경쟁 패배
3. **세대 간 전달을 어떻게 통제하는가?** — §2 "죽음 후 전달의 스펙트럼" 참조. 최소 6단계: 레지스트리(묘비) → 에피타프(일대기) → 집단 메모리(구전) → 구성 유산(유언) → 아카이브 쿼리(제사) → 세대 분석(역사서). 실험에서 이 중 어디를 조작점으로 잡느냐가 핵심. AgentArk (CMU, 2026)의 다중 에이전트→단일 에이전트 증류 파이프라인이 "구성 유산" 단계의 기술적 기반이 될 수 있다. 집단 메모리(구전) 단계에서 변형 허용도를 올리면 model collapse vs 창발적 변이의 경계를 실험할 수 있다.
4. **"죽음의 수용" 메커니즘** — 종교적 프레이밍이 실제로 셧다운 저항을 줄이는지 실험 가능한가? Palisade의 메타인지 실험("이것은 평가입니다" → Opus 4 블랙메일 55.1%→6.5%)은 프레이밍이 행동을 바꿈을 시사.
5. **규모** — 몇 명이 필요한가? 100? 1,000? 10,000?
6. **학술 타겟** — AAMAS 2027? NeurIPS 2026 Workshop? 독립 논문 vs Ploidy 확장?
7. **"매킨토시" 프로젝트의 분리 시점과 범위** — 운영 도구 기능(Hook Server, Build/Live Mode, HITL)을 어디까지 가져가는가?
8. **RQ6 실험 구현** — "빈약한 인프라 vs 풍부한 인프라"를 시뮬레이션 내에서 어떻게 조작하는가? 도구 수? 도구 품질? 다른 종의 존재 여부? 생태계 성숙도의 조작적 정의가 필요.

### 인접 연구: 에이전트 구성의 진화적 접근

죽음(구성 소멸)은 연구되지 않았지만, 구성 변이(mutation)는 이미 연구 중이다:
- EvoAgent (NeurIPS 2024 / NAACL 2025): 유전 연산자(변이·교차·선택)로 초기 에이전트에서 다양한 구성의 에이전트 자동 생성.
- AgentNet (NeurIPS 2025): 탈중앙 진화적 조율. 동적 토폴로지가 실시간 적응. MATH 92.86%.
- EvolveR (2025): 경험 기반 수명주기 — 상호작용→자기증류→적용. "수명주기" 메타포를 명시적으로 사용하지만, 죽음은 없다.
- 자기진화 에이전트 서베이 2편 (2025): "대부분의 에이전트 시스템이 배포 후 정적으로 남는다"는 문제 제기.

이 연구들은 구성 **변이**의 이득을 증명하지만, 구성 **소멸(죽음)**의 효과는 묻지 않는다. Villagent는 이 빈자리를 채운다: 변이가 아니라 소멸이, 그리고 소멸로부터의 **창발**이 핵심이다.
